## 一、系统是干什么的

这是一套 **AI 辅助的场外基金投资助手**，当前模式为 **semi_auto（半自动）**：

| 系统会做 | 系统不会做 |
|----------|------------|
| 每天拉取净值、算盈亏 | 不会在支付宝/天天基金自动下单 |
| 规则扫描（止损、仓位、换基冷却等） | 不保证 AI 建议一定正确 |
| AI 给出持有/加仓/减仓建议 | 不替你做最终投资决策 |
| QQ 邮件推送精简日报 | |
| 每周全市场选基 + 分批买入计划 | |
| 在线 HTML 完整报告（GitHub Pages） | |

**你的日常动作**：看邮件 → 点链接看完整报告 → 在支付宝手动确认申购/加仓。

## 二、每天自动发生什么（不用你管）

| 时间（北京时间） | 触发方式 | 你会收到 |
|------------------|----------|----------|
| **每天 20:35** | cron-job.org → GitHub `workflow_dispatch` | **精简持仓日报**邮件 |
| 同上（如有到期批次） | 读 `config/batch_state.json` | **分批买入提醒**邮件 |
| **每周日 10:00** | cron-job.org → 每周选基 workflow | **每周选基**邮件 |

定时由 [cron-job.org 外部触发](外部定时触发-cron-job.org.md)（比 GitHub 内置 schedule 更准时）。GitHub Actions 云端跑，**本机不用开机**。

## 三、三类邮件分别是什么

### 1. 精简持仓日报（每天）

- **止损/止盈/仓位** 规则扫描
- **趋势观察**：基金阶段涨跌判断（大涨小回/大跌小弹）
- **止盈策略**：分批止盈 + 动态回撤止盈 + 估值止盈（触发时展示）
- **AI 操作建议**（若有）
- 相关主题要闻（10 个关键词）
- 持仓净值、盈亏、仓位占比

**注意**：净值数据来自东财/AkShare，可能比支付宝晚 0～1 个交易日；**持仓操作以支付宝为准**。

### 2. 分批买入提醒（有到期批次时）

### 3. 每周全市场选基（每周日）

- **主题示例**：`【持仓】05-21 浮盈+9.0% · 持有`
- **正文包含**：结论一行、账户盈亏、沪深300对比、AI 摘要、持仓表、今日操作清单、规则提醒
- **底部**：`📊 点击查看完整报告`（在线链接）
- **附件**：`基金日报-YYYY-MM-DD.html`（离线也能看）

### 2. 分批买入提醒（按批次到期）

- **主题示例**：`【分批提醒】第2批 · 合计 1583 元`
- **何时发**：上次跑 `fund_recommend.py` 生成分批计划后，到第 7 天、第 14 天等节点
- **你要做**：按邮件明细去支付宝手动申购

### 3. 每周选基（周日）

- **主题示例**：`【选基】05-21 配置建议 · 预算 5000 元`
- **正文包含**：AI 配置思路、建议买入清单、下一批计划、270042 定投说明
- **底部/附件**：完整选基报告链接 + HTML 附件

## 四、配置文件速查（改哪里、什么时候改）

```
config/
├── positions.csv      ← 【最常改】真实持仓
├── fund_universe.csv  ← 关注池 / AI 候选基金
├── strategy.yaml      ← 策略、预算、分批、风控阈值
├── notify.yaml        ← 收件邮箱（一般不用动）
└── batch_state.json   ← 分批计划（系统自动写，一般不用手改）

.env                   ← API Key、QQ 邮箱授权码（本地私密，勿提交 Git）
```

### 4.1 `positions.csv` — 持仓（最重要）

**什么时候改**：买入、加仓、卖出、份额/成本变化后。

| 字段 | 含义 | 从哪里抄 |
|------|------|----------|
| fund_code | 6 位基金代码 | 支付宝基金详情页 |
| fund_name | 名称 | 可简写 |
| shares | 当前总份额 | 支付宝「持有份额」 |
| cost_per_share | 持仓成本价（加权平均） | 支付宝「持仓成本/成本价」，**不要填当日净值** |
| first_buy_date | 首次建仓确认日 | 第一次买入的 `YYYY-MM-DD`，用于 7 天赎回费判断 |
| last_buy_date | 最近一次买入日 | 最近加仓/申购确认日，用于 30 天换基冷却 |
| channel | 渠道 | 如 `支付宝` |

**新买一只基金** → 末尾新增一行。  
**对已有基金加仓** → 只改原行：`shares`、`cost_per_share`（用支付宝显示的加权成本）、`last_buy_date` 改为本次确认日；**`first_buy_date` 保持第一次建仓日不变**。

> 代码必须与支付宝完全一致，否则净值会对不上。

### 4.2 `fund_universe.csv` — 关注池

**什么时候改**：
- 想让某基金出现在日报「关注池」、参与 AI 选基
- 跑 `--sync-universe` 后系统自动追加推荐基金

| 字段 | 说明 |
|------|------|
| theme | 主题标签，如 `人工智能`、`宽基`、`医药` |
| risk_tag | `high` / `medium` / `low` |
| notes | 如 `仅定投`、`AI推荐-2026-05-21` |

已在 `positions.csv` 的基金会自动进日报；这里主要用于**未持仓但想跟踪**的基金。

### 4.3 `strategy.yaml` — 策略（偶尔改）

| 配置段 | 作用 | 常见修改 |
|--------|------|----------|
| `allocation` | 单基/主题仓位上限 | 账户上万后生效 |
| `signals` | 止损 -15%、止盈 15%/20%/30% 分批 | 按风险偏好调整 |
| `trailing_stop` | 动态回撤止盈（浮盈≥20% 后启用，回撤≥10% 触发） | 可调回撤阈值 |
| `valuation_take_profit` | 估值止盈（沪深300 PE 分位≥80%，默认关闭） | 按需开启 |
| `trend_observation` | 趋势观察（90日大涨小回/大跌小弹判断） | 可调阈值 |
| `trading.dca_only_funds` | 只定投、不加仓的基金 | 如 `270042` |
| `portfolio_plan` | 主仓、定投基金 | 换主仓时改 |
| `recommendation` | 选基预算、宽基比例、分批规则 | 换预算如 5000→8000 |

当前要点：
- 主仓：`012734`（人工智能）
- 定投：`270042` 每天 10 元，**不计入** 5000 预算
- 市值 < 1 万：放宽仓位上限检查（小账户模式）

### 4.4 `.env` — 密钥（首次配置 + 换 Key 时）

```powershell
copy .env.example .env
```

必填项：

```env
LLM_API_KEY=sk-...
LLM_BASE_URL=https://api.deepseek.com
LLM_MODEL=deepseek-chat

SMTP_USER=你的QQ邮箱
SMTP_PASSWORD=QQ邮箱授权码（不是登录密码）
NOTIFY_TO=收件邮箱

REPORT_PUBLIC_BASE_URL=https://stj005.github.io/investmentFund
```

GitHub Actions 需在仓库 **Settings → Secrets** 里配同样一套（不含 `.env` 文件本身）。

## 五、常见操作步骤

### 5.1 买入 / 加仓后更新持仓

1. 支付宝确认成交
2. 打开 `config/positions.csv`，按上表更新
3. 本地验证：

```powershell
cd d:\visual\makeMoney\investmentFund
python scripts/daily_report.py --no-email
```

4. 看终端输出和 `reports/YYYY-MM-DD.md`，与支付宝核对
5. 推送到 GitHub（否则云端邮件仍用旧数据）：

```powershell
git add config/positions.csv
git commit -m "chore: 更新持仓"
git push
```

### 5.2 手动看今天日报（不发邮件）

```powershell
python scripts/daily_report.py --no-email
```

输出位置：
- 本地 Markdown：`reports/YYYY-MM-DD.md`
- 在线 HTML：`docs/reports/YYYY-MM-DD.html`
- 操作清单：`data/checklist/YYYY-MM-DD.md`

### 5.3 手动发一封完整日报邮件

```powershell
python scripts/daily_report.py
```

### 5.4 强制刷新净值（不用缓存）

```powershell
python scripts/daily_report.py --no-email --cache   # 调试：用本地缓存加速
```

### 5.5 只看规则、不调 AI（省 API 费用）

```powershell
python scripts/daily_report.py --no-ai --no-email
```

### 5.6 全市场选基（有预算想加仓时）

```powershell
# 按 strategy.yaml 默认预算 5000
python scripts/fund_recommend.py

# 指定预算 + 写入关注池 + 发邮件
python scripts/fund_recommend.py --budget 5000 --sync-universe --email
```

报告：`reports/fund-recommend-YYYY-MM-DD.md`  
同时更新 `config/batch_state.json`（后续分批提醒靠它）。

选基规则（程序 + AI 共同约束）：
- 主仓 `012734` 必须加仓（默认 ≥1000 元）
- 宽基指数合计 ≥50%
- 分 3 批，每批间隔 7 天
- `270042` 继续日定投 10 元

### 5.7 在支付宝按邮件操作

1. 打开邮件 → 看「今日操作」或「分批提醒」
2. 支付宝 → 搜索基金代码 → 申购 / 加仓
3. 成交后回到 **5.1** 更新 `positions.csv`

## 十、常见问题

| 现象 | 处理 |
|------|------|
| 邮件持仓和支付宝不一致 | 改 `positions.csv` 并 `git push` |
| 邮件没有可点击链接 | 检查 `.env` 和 Actions 是否配了 `REPORT_PUBLIC_BASE_URL` |
| 链接 404 | Pages 是否启用；当天 workflow 是否已成功 commit `docs/reports/` |
| 没收到分批提醒 | 是否跑过选基；`batch_state.json` 是否在仓库里；是否到对应天数 |
| AI 建议很离谱 | 以规则扫描 + 支付宝数据为准；可 `--no-ai` 只看数据 |
| Actions 失败 | 看 Actions 日志；会尝试发失败告警邮件 |
| 基金代码对不上 | 以支付宝 6 位代码为准，检查 CSV 是否写错 |
| 日报净值日比支付宝晚 | 东财/AkShare 数据源可能比支付宝晚 0～1 天，**以支付宝为准** |
